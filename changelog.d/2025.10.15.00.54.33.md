# Fixed Kanban Column Underscore Normalization

## Problem
CLI commands and board generation used different column name normalization logic, causing mismatches when users typed column names with different formats (e.g., "in-progress" vs "in_progress").

## Root Cause
Two different `columnKey` functions existed:
- `packages/kanban/src/lib/kanban.ts` (preserved underscores)
- `packages/kanban/src/cli/command-handlers.ts` (only handled spaces)

## Solution
- Exported `columnKey` from `kanban.ts` and removed duplicate from `command-handlers.ts`
- Updated normalization logic to convert spaces and hyphens to underscores: `.replace(/[\s-]+/g, '_')`

## Impact
All column name variants now correctly access the same column:
- `"in_progress"` → 8 tasks ✅
- `"in-progress"` → 8 tasks ✅  
- `"in progress"` → 8 tasks ✅

## Files Modified
- `packages/kanban/src/lib/kanban.ts`
- `packages/kanban/src/cli/command-handlers.ts`
