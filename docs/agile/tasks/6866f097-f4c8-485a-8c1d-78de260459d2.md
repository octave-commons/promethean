# Implement LLM-powered kanban explain command

## Acceptance Criteria
- [ ] Add `pnpm kanban explain` command to CLI
- [ ] Integrate with Ollama LLM for intelligent explanations
- [ ] Explain task status, transitions, and workflow rules
- [ ] Provide recommendations for blocked/stuck tasks
- [ ] Support natural language queries about board state
- [ ] Add caching for LLM responses to improve performance
- [ ] Include error handling for LLM unavailability

## Implementation Details
1. **CLI Extension**: Add explain subcommand to kanban CLI
2. **LLM Integration**: Connect to Ollama API with proper prompts
3. **Context Builder**: Gather board state, task history, and rules
4. **Response Parser**: Structure LLM responses for CLI output
5. **Caching Layer**: Implement response caching with TTL
6. **Error Handling**: Graceful fallback when LLM unavailable
7. **Testing**: Unit tests for all components

## Dependencies
- Ollama service running and accessible
- Existing kanban CLI structure
- LLM prompt engineering for board explanations

- Caching & Error Handling: 1 day
