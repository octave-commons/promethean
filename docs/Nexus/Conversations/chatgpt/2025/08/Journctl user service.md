> If you want to look at logs for a **user service** (i.e. one managed by `systemd --user`), the syntax for `journalctl` changes slightly compared to system-wide services. Here’s the breakdown:
> 
> ---
> 
> ### 1. View logs for a user service
> ```bash
> journalctl --user -u <service-name>.service
> ```
> Example:
> ```bash
> journalctl --user -u mybot.service
> ```
> 
> ---
> 
> ### 2. Follow logs live (like `tail -f`)
> ```bash
> journalctl --user -u mybot.service -f
> ```
> 
> ---
> 
> ### 3. Limit how much history you see
> - Last 100 lines:
>  ```bash
>  journalctl --user -u mybot.service -n 100
>  ```
> - Since boot:
>  ```bash
>  journalctl --user -u mybot.service -b
>  ```
> - Since a specific time:
>  ```bash
>  journalctl --user -u mybot.service --since "2025-08-30 09:00:00"
>  ```
> 
> ---
> 
> ### 4. If you don’t see any logs
> Sometimes user services don’t log unless you explicitly set `StandardOutput=journal` and/or `StandardError=journal` in the unit file (under `[Service]`). For example:
> ```ini
> [Service]
> ExecStart=/home/err/bin/mybot
> StandardOutput=journal
> StandardError=journal
> ```
> Then reload with:
> ```bash
> systemctl --user daemon-reload
> systemctl --user restart mybot.service
> ```
>

> 
> Use the exact unit name you see in `systemctl --user status`:
> 
> ```bash
> journalctl --user -u ytwall.service
> journalctl --user -u ytwall.service -f # live follow
> journalctl --user -u ytwall.service -n 200 # last 200 lines
> ```
> 
> If you ever forget the exact name, list and grep:
> ```bash
> systemctl --user list-units --type=service --all | grep -i yt
> ```
> 
> Other reliable ways to pull the logs:
> 
> - By main PID (from your status output, `Main PID: 9488`):
>  ```bash
>  journalctl --user _PID=9488
>  ```
> 
> - Wildcard the unit:
>  ```bash
>  journalctl --user --unit 'ytwall*'
>  ```
> 
> - If you set a `SyslogIdentifier=` in the unit, you can filter by that too:
>  ```bash
>  journalctl --user SYSLOG_IDENTIFIER=ytwall
>  ```
> 
> If you still see nothing (unlikely, since `systemctl status` is already showing log lines), check the unit has:
> ```ini
> [Service]
> StandardOutput=journal
> StandardError=journal
> ```

> ```bash
> systemctl --user daemon-reload
> systemctl --user restart ytwall.service
> ```
>
