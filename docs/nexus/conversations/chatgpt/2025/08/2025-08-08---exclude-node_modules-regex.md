>[!nexus_agent] **Assistant** - 08/08/2025 at 8:54 PM
> If you just want a regex that matches anything **not** in a `node_modules` folder so you can use it in Obsidian’s *“Exclude files”* or similar ignore settings, you can use a **negative lookahead**. 
> 
> For example: 
> 
> ```regex
> ^(?!.*\/node_modules\/).*
> ```
> 
> **Explanation:**
> - `^` → start of string
> - `(?!.*\/node_modules\/)` → negative lookahead: ensures the path does **not** contain `/node_modules/`
> - `.*` → match the rest of the line (any file path)
> 
> This works for both:
> - `src/node_modules/foo.js`
> - `node_modules/foo/bar.js`
> 
> They won’t match, but `src/utils/bar.js` will.
> 
> ---
> 
> If Obsidian is on Windows and your paths use backslashes, swap `/` for `\\`:
> 
> ```regex
> ^(?!.*\\node_modules\\).*
> ```
> 
> ---
> 
> Do you want this regex to also block the **top-level `node_modules` folder** and **nested ones** in subdirectories? That changes the boundary handling slightly.

> Ah — so you want a regex that **only matches** things inside a `node_modules` folder. 
> That way, Obsidian can use it in the "Exclude files" list to filter them out. 
> 
> For Unix-style `/` paths: 
> ```regex
> .*/node_modules/.* 
> ```
> 
> For Windows `\` paths: 
> ```regex
> .*\\node_modules\\.* 
> ```
> 
> **Explanation:**
> - `.*` → anything before 
> - `/node_modules/` (or `\\node_modules\\`) → folder name with slashes around it 
> - `.*` → anything after 
> 
> This will match: 
> ```
> node_modules/foo.js
> src/node_modules/bar/index.js
> packages/pkg-a/node_modules/pkg-b/file.js
> ```
> 
> And won’t match anything outside `node_modules`. 
> 
> ---
> 
> If your Obsidian vault is cross-platform, you can make it portable: 
> ```regex
> .*[\\/]+node_modules[\\/]+.*
> ```
> That matches either `/` or `\` as the path separator. 
>
